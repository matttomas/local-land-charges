Search History Table

User story

As a citzen/conveyancer
I want to be able to quickly find a search i have done in the past
So that i can easily view or repeat it

Old design

  • user found it difficult to distinguish where one search starts and ends
  • users struggled to find the drop down and where confused what it did when they did
  • users struggled to find the search they were intrested in using the title number only
  • the table was too congested and full of unesccary infomration
  • the users cares about what date they have until to repeat (ASSUMPTION)

New design

  • every other search is shaded grey
  • change plus to down arrow and relocated for expansion
  • user inputted search area description in left hand column rather than title number
  • don't list number of repeats
  • list expiry date

Resarch Findings

  • All users could easily find the charge they were intrested in
  • All users used the address/serarch area description to find the search they are intrested in
  • All users were able to repeat a search
  • Alot of users struggled to find where they would repeat a search from and needed prompting to where to log in
